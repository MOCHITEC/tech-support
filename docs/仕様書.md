# 会議室予約システム 仕様書

本書はシステムの**正式仕様(ground truth)**である。トリアージエージェントは、ユーザ報告の挙動が本書と乖離していれば「バグ」、本書に存在しない振る舞いを求めるものであれば「機能要望」と判定する。

## 1. エンティティ

| エンティティ | 属性 |
|---|---|
| Room(会議室) | `id`, `name`, `hourly_rate`(1時間あたりの料金, 円) |
| User(利用者) | `id`, `name`(デモ用ユーザ) |
| Reservation(予約) | `id`, `room_id`, `user_id`, `start_time`, `end_time`, `status`(`active` / `cancelled`), `created_at` |

## 2. 料金計算

- 予約料金 = `hourly_rate` × 利用時間(時間単位)。
- **利用時間が 2 時間以上の場合、合計金額に 10% の割引を適用する。**
- 端数は円未満を切り捨てる。

例(`hourly_rate` = 1,000 円):

| 利用時間 | 計算 | 料金 |
|---|---|---|
| 1 時間 | 1,000 × 1 | 1,000 円 |
| 2 時間 | 1,000 × 2 × 0.9 | 1,800 円 |
| 3 時間 | 1,000 × 3 × 0.9 | 2,700 円 |

## 3. 予約作成

- `end_time` は `start_time` より後でなければならない。満たさない場合は作成を拒否する。
- **同一会議室で時間帯が重複する予約は作成できない(二重予約の禁止)。** 重複とは、対象会議室の既存の `active` 予約と区間 `[start_time, end_time)` が交差することを指す(境界が接するだけ、例: 既存予約の `end_time` と新規予約の `start_time` が同一、は重複ではない)。
- 上記を満たす場合のみ、`status = active` の予約を作成する。

## 4. キャンセル

- `active` の予約のみキャンセルできる。既に `cancelled` の予約は再キャンセルできない。
- **開始時刻が現在時刻より後(未来)の予約のみキャンセルできる。** 開始時刻が現在時刻以前(過去または進行中)の予約はキャンセルできない。
- キャンセルすると `status = cancelled` になる。

## 5. 認証

- デモ用の疑似ログインのみ。セッション開始時に専用デモユーザが自動割当される(ユーザ選択画面は設けない)。
- 利用者は自分が作成した予約のみ閲覧・キャンセルできる。
