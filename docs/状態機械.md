# チケット状態機械

フィードバックチケットの状態と許可遷移の正式定義。アプリ・エージェント・ステータスページはこの表のみに従う。許可されない遷移はコードで拒否する。

## 状態一覧

| 状態(コード) | 表示名 | 区分 |
|---|---|---|
| `RECEIVED` | 受付 | 初期 |
| `TRIAGING` | トリアージ中 | 進行 |
| `REPRODUCING` | 再現確認中 | 進行 |
| `AWAITING_INFO` | 追加情報待ち | 進行(ユーザ待ち) |
| `FIXING` | 修正中 | 進行 |
| `AWAITING_REVIEW` | レビュー待ち | 進行(IT待ち) |
| `RELEASED` | リリース済み | 終了(成功) |
| `FEATURE_REQUEST` | 要望としてIT検討中 | 終了(要望) |
| `DUPLICATE` | 重複候補(IT確認待ち) | 終了(IT判断) |
| `ESCALATED` | 人間へエスカレーション | 終了(IT判断) |
| `UNFIXABLE` | 修正不能 | 終了(失敗) |
| `PR_REJECTED` | PR却下 | 終了(IT判断) |
| `DEPLOY_FAILED` | デプロイ失敗 | 進行(再試行可) |

## 許可遷移

| From | To(許可先) |
|---|---|
| `RECEIVED` | `TRIAGING` |
| `TRIAGING` | `REPRODUCING`, `FEATURE_REQUEST`, `DUPLICATE`, `ESCALATED` |
| `REPRODUCING` | `FIXING`, `AWAITING_INFO`, `ESCALATED` |
| `AWAITING_INFO` | `REPRODUCING`, `ESCALATED` |
| `FIXING` | `AWAITING_REVIEW`, `ESCALATED`, `UNFIXABLE` |
| `AWAITING_REVIEW` | `FIXING`, `RELEASED`, `PR_REJECTED`, `DEPLOY_FAILED` |
| `DEPLOY_FAILED` | `FIXING`, `ESCALATED` |
| `RELEASED` | (なし) |
| `FEATURE_REQUEST` | (なし) |
| `DUPLICATE` | (なし) |
| `ESCALATED` | (なし) |
| `UNFIXABLE` | (なし) |
| `PR_REJECTED` | (なし) |

## 補足

- `AWAITING_INFO` と `AWAITING_REVIEW` は外部(ユーザ / IT)入力待ちで、自動では進まない。
- 終了状態からの遷移は一切許可しない(再オープンは新規チケットとして扱う)。
- `DEPLOY_FAILED` のみ終了させず `FIXING` への再試行を許可する。
